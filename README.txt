진짜 works 페이지 수정본 - 적용 방법
=========================================

찾은 것
실제 사이트는 React(Vite)로 만들어져 있고, /works 라우트가 있어요.
빌드된 JS 파일(index-CQHrhn9j.js) 안에 시공사례 데이터가 그대로
하드코딩되어 있었어요 (Fh 라는 배열). 그 안의 10개 항목이 전부
"패브릭 부스 시공 / COEX" 같은 가짜(placeholder) 제목과 존재하지도
않는 이미지 파일(1.jpg~10.jpg, work7~10)을 가리키고 있었어요.
→ 이게 이번에 발견하신 "우리가 시공 안 한 이미지" 정체였습니다.

한 것
- 그 데이터 배열을 실제 채팅으로 올려주신 26장 사진 기반 데이터로
  통째로 교체했어요 (제목/장소/날짜 전부 실제 정보로).
- 코드 나머지(레이아웃, 호버 효과, MAJOR VENUES 섹션 등)는 원본 그대로
  건드리지 않았어요. node --check 로 문법 검증도 통과했습니다.
- CSS 파일(index-Cy9bIrve.css)은 수정 안 했으니 그대로 쓰시면 됩니다.

적용 방법 (emsystem.io 깃허브 레포 기준)
1. 이 zip 안의 assets/index-CQHrhn9j.js 로
   레포의 assets/index-CQHrhn9j.js 파일을 덮어쓰세요. (파일명 동일)
2. 이 zip 안의 images/works/ 폴더 전체를
   레포의 images/ 폴더 안에 그대로 복사해 넣으세요.
   (기존 images/1.jpg~12.jpg, work7~10 등 가짜 이미지는
    이제 아무 데서도 참조하지 않으니 그대로 두거나 지우셔도 무방)
3. index.html, index-Cy9bIrve.css는 그대로 두시면 됩니다.
4. 깃허브에 커밋 & 푸시하면 emsystem.co.kr/works 에 바로 반영돼요.
